## Systems Across Boundaries

I work on software systems that need to cross boundaries: between platforms, runtimes, languages, file formats, older technologies, and very specific human workflows. My professional focus is cross-platform application and game development, legacy modernization, specialized business tools, service development, plugin architectures, API surfaces, and systems where reusable logic matters more than platform-specific ceremony.

My current explorations are moving deeper into programming language design, embeddable scripting runtimes, binary data manipulation, compression, and AI-assisted architecture. I use AI as an architectural and implementation assistant, while keeping system direction, taste, judgment, and responsibility human-led.

## Working Areas

- Cross-platform applications, 2D games, browser games, and mobile systems.
- Ports from abandoned or rigid platforms into modern, maintainable runtimes.
- Specialized tools that turn narrow business workflows into reliable software.
- CLIs, services, custom protocols, build systems, and developer infrastructure.
- Plugin systems, API contracts, embeddable scripting, and language-runtime interfaces.
- Binary data, file formats, compression, transformation, and low-level representation.

## Operating Principles

- Prefer portable logic over platform silos.
- Treat APIs and plugins as contracts, not afterthoughts.
- Keep old systems understandable before replacing them.
- Design tools around real workflows, not generic abstractions.
- Use AI as an amplifier while preserving human-led architecture.
